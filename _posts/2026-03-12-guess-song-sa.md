## 一、系統概述

猜歌遊戲（GuessSong）是 XPG Server 平台的第三款遊戲內容，屬於益智互動型多人遊戲。玩家聆聽 30 秒音樂片段後，透過點擊字元拼出歌名，依計分模式決定勝負。

---

## 二、使用者角色

| 角色 | 說明 |
|------|------|
| **玩家（Player）** | 登入後可進行個人或多人模式 |
| **遊客（Tourist）** | 僅可觀看，不可進行遊戲 |
| **房主（Room Owner）** | 創建房間的玩家，可設定模式、開始遊戲 |
| **管理員（Admin）** | 透過後台白名單 IP 執行管理命令 |

---

## 三、遊戲模式需求

### 3.1 個人模式（Solo）

- 玩家進入後立即開始，無需等待他人
- 可設定遊戲輪數
- 完成後顯示個人總分與歷史紀錄比較
- 房間為臨時房，遊玩完畢後自動銷毀

### 3.2 多人模式（Multi）

- 房主創房後取得 6 碼英數房間代碼
- 其他玩家輸入代碼 + 暱稱後加入等待室
- 房主確認人數後手動開始遊戲
- 個人計分排行

### 3.3 隊伍模式（Team）

- 房主設定隊伍數量（2 ～ 5 隊）及隊伍名稱（例：各隊隊長名字）
- 創房時選擇人員分配方式：
  - **自由選隊**：玩家自行選擇隊伍，等待室顯示各隊目前人數
  - **系統平均分配**：加入後自動分配至人數最少的隊伍，同人數時隨機選一
- 支援最多 100 人同時遊玩
- 個人得分累計至所屬隊伍，最終以隊伍總分決勝負

## 四、功能需求

### 4.1 房間管理

| 編號 | 需求 |
|------|------|
| F-01 | 系統可創建個人、多人、隊伍三種房間 |
| F-02 | 多人 / 隊伍模式需產生唯一 6 碼英數房間代碼 |
| F-03 | 玩家可憑代碼加入房間 |
| F-04 | 隊伍模式玩家加入時可輸入暱稱並選擇或被分配隊伍 |
| F-05 | 等待室每 5 秒廣播一次各隊人數與總人數 |
| F-06 | 房主可隨時開始遊戲（最少 1 人以上）|
| F-07 | 房主離開後房間自動關閉，所有玩家返回大廳 |

### 4.2 題目系統

| 編號 | 需求 |
|------|------|
| F-08 | 題目資料由後端 API 取得（`Song_ConnString`）|
| F-09 | 開發期使用 mockJSON 模擬題庫 API |
| F-10 | 每輪推送一首歌曲資訊：歌名字元列表、試聽連結、封面圖 |
| F-11 | 字元列表包含答案字元 + 隨機干擾字元，總數為答案長度 × 3（最少 12、最多 24）|
| F-12 | 字元在 Client 端隨機排列顯示 |
| F-13 | 題庫資料來源控制為純中文歌名，不含英文或數字 |

### 4.3 作答機制

| 編號 | 需求 |
|------|------|
| F-14 | 玩家點擊字元依序拼出歌名 |
| F-15 | 點擊已選字元可取消並歸回選字區 |
| F-16 | 完成後送出答案，或倒數計時結束時自動提交 |
| F-17 | 每輪每人只能提交一次答案 |
| F-18 | Server 收到作答後先暫存記憶體，時間到後統一結算 |

### 4.4 計分系統

| 編號 | 需求 |
|------|------|
| F-19 | 創房時設定計分模式（可複選）：答對加分、速度加分 |
| F-20 | **答對加分**：答對固定得 100 分 |
| F-21 | **速度加分**：答對後依作答時間計分，滿分 100，每秒遞減，答錯不得分 |
| F-22 | **兩者疊加**：答對 100 分 + 時間分（最高 200 分）|
| F-23 | 隊伍模式：個人得分自動累計至所屬隊伍 |

### 4.5 結算與報表

| 編號 | 需求 |
|------|------|
| F-24 | 每輪結束推送：答案、各玩家得分、隊伍排行（隊伍模式）|
| F-25 | 最終結算推送：個人總分排行、隊伍總分排行（隊伍模式）|
| F-26 | 遊戲結束後寫入三層報表（將報表、局報表、個人報表）|

---

## 五、非功能需求

| 編號 | 需求 |
|------|------|
| NF-01 | 隊伍模式支援最多 100 人同時連線 |
| NF-02 | 廣播採聚合推送（batch），避免大人數時效能瓶頸 |
| NF-03 | 個人排行榜只顯示自己名次，不展開全部 100 人列表 |
| NF-04 | 遊戲房狀態應可從任意場景中斷（系統維護、關機）並優雅處理 |
| NF-05 | 所有命令回應時間 < 500ms（正常網路環境）|

## 六、使用流程

### 6.1 個人模式流程

```
玩家登入
  → 選擇「猜歌遊戲」→ 選擇「個人模式」
  → 設定輪數 → 進入房間 → 立即開始
  → [輪次循環] 播放試聽 → 作答 → 顯示結果
  → 遊戲結束 → 顯示總分 → 返回大廳
```

### 6.2 多人模式流程（房主）

```
玩家登入
  → 選擇「猜歌遊戲」→「建立多人房」
  → 設定輪數、計分模式 → 取得房間代碼（如：R7X2KP）
  → 等待室（顯示目前人數）
  → 按「開始遊戲」
  → [輪次循環] → 遊戲結束 → 最終排行
```

### 6.3 多人模式流程（參與者）

```
玩家登入
  → 選擇「猜歌遊戲」→「加入房間」
  → 輸入代碼 R7X2KP → 進入等待室
  → 等待房主開始 → [輪次循環] → 遊戲結束
```

### 6.4 隊伍模式流程（房主）

```
玩家登入
  → 選擇「猜歌遊戲」→「建立隊伍房」
  → 設定隊伍數（如3隊）、隊伍名稱、分配模式、計分
  → 取得房間代碼 → 等待室（顯示各隊人數）
  → 確認人數平衡 → 按「開始遊戲」
```

### 6.5 隊伍模式流程（參與者）

```
玩家登入
  → 輸入代碼 → 輸入暱稱
  ├─ 自由選隊：顯示各隊人數 → 點選隊伍 → 進入等待室
  └─ 系統分配：自動分配 → 顯示「你已被分配到：XX隊」→ 進入等待室
```

## 七、WebSocket 命令協定

### 命令格式

```json
{ "cmd": "命令字串", "data": { ...命令資料... } }
```

### 大廳 / 房間管理命令

| 命令 | 方向 | 說明 | 備註 |
|------|------|------|------|
| `gscr` | C→S / S→C | 創建猜歌房 | 回傳房間代碼 |
| `gsjr` | C→S / S→C | 以代碼加入房間 | 含暱稱、隊伍選擇 |
| `gsgi` | S→C | 房間完整資訊 | 進房後收到 |
| `gsts` | S→C | 等待室隊伍人數更新 | 每 5 秒廣播 |
| `gsst` | C→S | 房主開始遊戲 | |

### 遊戲中命令

| 命令 | 方向 | 說明 | 備註 |
|------|------|------|------|
| `gsq`  | S→C | 推送題目（字元列表 + 試聽連結）| 每輪開始時 |
| `gsa`  | C→S / S→C | 玩家提交答案 | 回傳 code |
| `gsrr` | S→C | 本輪結果（答案、個人分、隊伍排行）| |
| `gsfr` | S→C | 遊戲最終結算 | |

## 八、命令資料格式

### `gscr` 創建猜歌房

**Client → Server（請求）**

```json
{
  "cmd": "gscr",
  "data": {
    "roomMode": 3,
    "scoreMode": 3,
    "roundCount": 10,
    "roundTimeSec": 30,
    "maxPlayers": 100,
    "teamCount": 3,
    "teamNames": ["小明隊", "小華隊", "小美隊"],
    "assignMode": 2
  }
}
```

**Server → Client（回應）**

```json
{
  "cmd": "gscr",
  "data": {
    "code": 0,
    "roomCode": "R7X2KP",
    "roomMode": 3,
    "teams": [
      { "index": 0, "name": "小明隊", "count": 0 },
      { "index": 1, "name": "小華隊", "count": 0 },
      { "index": 2, "name": "小美隊", "count": 0 }
    ]
  }
}
```

| code | 說明 |
|------|------|
| 0 | 成功 |
| 1 | 參數錯誤 |
| 2 | 玩家不在大廳 |

### `gsjr` 加入房間

**Client → Server（請求）**

```json
{
  "cmd": "gsjr",
  "data": {
    "roomCode": "R7X2KP",
    "nickname": "阿明",
    "teamIndex": 1
  }
}
```

> `teamIndex`：自由選隊時填入（0-based），系統平均分配時填 -1

**Server → Client（回應）**

```json
{
  "cmd": "gsjr",
  "data": {
    "code": 0,
    "assignedTeam": 1,
    "teamName": "小華隊"
  }
}
```

| code | 說明 |
|------|------|
| 0 | 成功 |
| 1 | 代碼不存在 |
| 2 | 房間已滿 |
| 3 | 遊戲已開始 |

### `gsgi` 房間完整資訊

**Server → Client（進房後推送）**

```json
{
  "cmd": "gsgi",
  "data": {
    "roomCode": "R7X2KP",
    "roomMode": 3,
    "scoreMode": 3,
    "roundCount": 10,
    "roundTimeSec": 30,
    "assignMode": 2,
    "isHost": 0,
    "myTeamIndex": 1,
    "teams": [
      { "index": 0, "name": "小明隊", "count": 4 },
      { "index": 1, "name": "小華隊", "count": 3 },
      { "index": 2, "name": "小美隊", "count": 3 }
    ],
    "totalPlayers": 10
  }
}
```

### `gsts` 等待室人數更新

**Server → Client（每 5 秒廣播）**

```json
{
  "cmd": "gsts",
  "data": {
    "teams": [
      { "index": 0, "name": "小明隊", "count": 32 },
      { "index": 1, "name": "小華隊", "count": 28 },
      { "index": 2, "name": "小美隊", "count": 30 }
    ],
    "total": 90
  }
}
```

### `gsq` 推送題目

**Server → Client（每輪開始）**

```json
{
  "cmd": "gsq",
  "data": {
    "round": 3,
    "totalRounds": 10,
    "previewUrl": "https://audio-ssl.itunes.apple.com/...preview.m4a",
    "artworkUrl": "https://is1-ssl.mzstatic.com/image/...100x100.jpg",
    "artistName": "周杰倫",
    "releaseYear": 2003,
    "chars": ["情","書","愛","你","我","月","小","天","的","心","夜","想","花","給","月","幸","運","他","她","了"],
    "answerLength": 2,
    "timeSec": 30
  }
}
```

> `chars`：字元列表，Client 端自行隨機排列顯示
> `answerLength`：答案字數，供 Client 顯示輸入格數

### `gsa` 提交答案

**Client → Server（玩家作答）**

```json
{
  "cmd": "gsa",
  "data": {
    "round": 3,
    "answer": "情書"
  }
}
```

**Server → Client（確認回應）**

```json
{
  "cmd": "gsa",
  "data": {
    "code": 0,
    "round": 3
  }
}
```

| code | 說明 |
|------|------|
| 0 | 成功收到 |
| 1 | 非作答階段 |
| 2 | 已提交過 |

### `gsrr` 本輪結果

**Server → Client（時間到後廣播）**

```json
{
  "cmd": "gsrr",
  "data": {
    "round": 3,
    "answer": "情書",
    "artistName": "周杰倫",
    "myResult": {
      "isCorrect": true,
      "answerTime": 8.3,
      "roundScore": 175,
      "totalScore": 450
    },
    "teamRank": [
      { "index": 0, "name": "小明隊", "totalScore": 1250 },
      { "index": 2, "name": "小美隊", "totalScore": 1100 },
      { "index": 1, "name": "小華隊", "totalScore": 980 }
    ]
  }
}
```

### `gsfr` 最終結算

**Server → Client（遊戲結束）**

```json
{
  "cmd": "gsfr",
  "data": {
    "myRank": 5,
    "myScore": 1250,
    "teamRank": [
      { "index": 0, "name": "小明隊", "totalScore": 12500, "rank": 1 },
      { "index": 2, "name": "小美隊", "totalScore": 11000, "rank": 2 },
      { "index": 1, "name": "小華隊", "totalScore": 9800,  "rank": 3 }
    ],
    "topPlayers": [
      { "nickname": "阿明", "teamName": "小明隊", "score": 1800 },
      { "nickname": "小花", "teamName": "小美隊", "score": 1700 }
    ]
  }
}
```

## 十、mockJSON 題庫格式（開發期）

開發期由 `GuessSongMockDb` 提供固定題庫，格式如下：

```json
[
  {
    "id": 1,
    "trackName": "情書",
    "artistName": "周杰倫",
    "releaseYear": 2003,
    "previewUrl": "https://example.com/preview1.m4a",
    "artworkUrl": "https://example.com/artwork1.jpg"
  },
  {
    "id": 2,
    "trackName": "小幸運",
    "artistName": "田馥甄",
    "releaseYear": 2015,
    "previewUrl": "https://example.com/preview2.m4a",
    "artworkUrl": "https://example.com/artwork2.jpg"
  }
]
```

> mockJSON 至少準備 **30 首**以上，確保測試時不重複出題。
